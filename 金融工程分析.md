### 1.3 权益端 (隐含净值)

系统不发行治理代币，不存在传统的股东权益。系统的**隐含净值** = 资金池余额 + 应收追偿款 - 坏账准备金 - 已承兑债务票据。明链 `PoolBalance` 是该净值的链上代理变量。

### 2.2 nekop2p 的信用乘数

系统通过**社交资本化**实现无需超额抵押的信用创造：

```
nekop2p 信用创造模型:
  ┌──────────────────────────────────────────────────────────────┐
  │                                                               │
  │  担保人质押 Bond = 1000                                       │
  │    ↓                                                          │
  │  被担保人获得信用额度 ≤ 1000 × 信用系数                        │
  │    ↓                                                          │
  │  被担保人实际借贷 = 800 (占用)                                 │
  │    ↓                                                          │
  │  被担保人的下级担保人继续获得额度 ≤ 800 × 系数                 │
  │    ↓                                                          │
  │  递归...                                                      │
  │                                                               │
  │  一层原始 Bond 1000 → 系统内可衍生总信用:                      │
  │    1000 × (1 + c + c² + c³)  (c = 信用系数 < 1)              │
  │                                                               │
  │  类比: 商业银行的存款准备金-贷款乘数效应                        │
  │  原始存款 → 贷款 → 派生存款 → 再贷款 → ...                    │
  └──────────────────────────────────────────────────────────────┘
```

### 2.3 信用乘数公式

设定：
- $B_0$ = 初始担保质押金
- $c \in (0, 1)$ = 信用传递系数（由担保人信任权重决定）
- $d \le 3$ = 邓巴数信任链最大深度

则系统内由 $B_0$ 衍生的总信用流动性为：

$$L(B_0) = B_0 \cdot \sum_{i=0}^{d} c^i = B_0 \cdot \frac{1 - c^{d+1}}{1 - c}$$

**关键参数**：
- $c$ 越小 → 信用扩张越保守 → 资金利用率低但风险可控
- $c$ 越大 → 信用扩张越激进 → 资金利用率高但系统性风险升高
- $d$ 为硬上限 (邓巴数3)，自然约束乘数不会无限放大

### 2.4 递归追偿的风险分层 (Tranching)

违约风险沿着担保树进行**分级切片**：

```
违约事件:
  借款人违约 (金额 X)
       │
       ▼
  [优先级1] 借款人链上匿名资产 ──→ 直接扣划
       │ (不足)
       ▼
  [优先级2] 直接担保人 Bond ──→ 全额罚没至覆盖
       │ (不足)
       ▼
  [优先级3] 上级推荐人 Bond ──→ 递归上溯
       │ (不足)
       ▼
  [优先级4] 全局坏账准备金 ──→ 最终兜底
```

从金融工程角度看，这等同于将违约风险通过担保树进行了**优先级分层 (Senior/Mezzanine/Junior Tranche)**：
- **Junior**: 借款人自身资产 (最先吸收损失)
- **Mezzanine**: 直接担保人 (次级吸收)
- **Senior**: 上级担保人 + 全局准备金 (最后吸收)

这种设计迫使每个担保人充当「民间风控员」——微观审慎监管被分散到网络每个节点。

## 三、经济循环：三路分账与交叉补贴

### 3.1 三路分账模型

明域游戏和交易所的每笔手续费在链上自动执行三路分账：

```
  交易手续费 (feeAmount)
       │
       ├──→ 作者分成 (AuthorShare)
       │       ↓
       │    激励内容生产者持续创新
       │
       ├──→ 服务器运营 (ServerShare)
       │       ↓
       │    补贴承担物理计算与网络带宽的节点
       │
       └──→ 生态基金/坏账准备金 (Foundation/Pool Fee)
               ↓
            维护公地 + 对冲暗域清算风险 + 支付节点工资
```

```go
// 实际实现: app/app.go RecordGameTx
authorFee := feeAmount * game.AuthorShare / 100      // 作者分成
serverFee := feeAmount * game.ServerShare / 100      // 服务器分成
poolFee   := feeAmount - authorFee - serverFee       // 生态基金兜底
```

**分账比率约束**：
- `AuthorShare + ServerShare ≤ 100`（池费 = 余量）
- 游戏作者在发布时设定比率，链上不可篡改
- 各游戏的分账比率独立，形成差异化竞争

### 3.3 节点劳动力市场的经济约束

```
全网正式节点数 ≤ 资金池月均收入 ÷ 单个节点最低激励工资

节点数由 GDP 决定，不是人为拍板。
```

这构成了系统的**内生货币纪律**——节点膨胀只能由经济活动增长支撑，防止了大多数 PoS 链的「通胀养节点」陷阱。

## 四、系统性金融风险

### 4.1 风险矩阵

| 风险类别 | 严重度 | 触发条件 | 影响范围 | 当前防御 |
|:---|:---|:---|:---|:---|
| **担保链多米诺 (Liquidity Contagion)** | 🔴 严重 | 核心节点恶意违约/物理灭失 | 整条树状分支 | 递归追偿 (火上浇油) |
| **Inkwell 隐形通胀 (Liquidity Discount)** | 🟠 重要 | 大量资金被锁 1~90 天 | 暗域资产折价 | 用户自主选择 |
| **明域通货紧缩 (Deflation Trap)** | 🟠 重要 | 掉落上限 P 过于激进 | 明域游戏经济 | 算法参数可治理调整 |
| **摆渡节点丢失 (Ferry Loss)** | 🟡 中等 | 物理设备损坏 | 单笔影子收据悬空 | 无超时回滚机制 |
| **社区级雪崩 (Community Avalanche)** | 🔴 严重 | 外交通道违约 + 递归 Slashing | 整个 Namespace | 外交通道 Bond 质押 |
| **社交图中心化 (Centralization)** | 🟡 中等 | 少数节点担保过多 | 去中心化假设瓦解 | 邓巴数硬上限 |

### 4.2 担保链多米诺骨牌效应 (深度分析)

**场景设定**：
- 核心节点 X 担保了 15 个下级，累计担保额 = 10,000
- 节点 X 的上级担保人 Y 再担保了 5 个节点，担保额 = 8,000
- X 的设备被执法部门没收，其 Bond 无法追回

**连锁反应**：
```
X 违约 10,000
   ↓
[层1] X 的 Bond 被全额罚没: 覆盖 3,000 → 剩余坏账 7,000
   ↓
[层2] X 的 15 个下级被触发连带追偿: 每人被扣 200~500 →
       其中 3 个下级因保证金不足同时破产
   ↓
[层3] Y (X 的上级) 被追偿: Bond 扣除 4,000 → Y 的 5 个下级触发连带
   ↓
[层4] 递归继续 → 局部信用瞬间雪崩
   ↓
[后果] 该分支用户发现「我的保证金莫名其妙被扣了」
       → 恐慌性撤保 → 信用供给收缩 → 去中心化银行挤兑 (Bank Run)
```

**风险本质**：递归追偿在正常情况下是有效的威慑机制，但在极端事件中它是**风险放大器**——将单点故障指数级扩散。

### 4.3 Inkwell 延迟池的资产折价

暗域 `Inkwell` 为了模糊资金追踪，将清算资金强制拆分并延迟 1~90 天释放：

$$\text{暗域资产实际价值} = \text{名义价值} \times D(t, r)$$

其中 $D(t, r)$ 是流动性折价因子，$t$ 为锁定天数，$r$ 为明域同期无风险收益率。当 $t \to 90$ 天时，$D$ 可能显著低于 1。

**货币银行学视角**：
- 资金周转速度 (Velocity of Money) 被人为压低
- 市场剧烈波动时，锁仓资产无法对冲 → **隐形机会成本**
- 长期可能形成「暗域折价」对「明域溢价」的双轨市场

### 4.4 明域通货紧缩陷阱

游戏掉落算法的全局上限 $P$ 动态调节产出：

$$E_i = f(K_i) \cdot \min(P, \sum \text{已产出})$$

若 $P$ 调节过激：
1. 用户激增 → 单用户可分配份额 $\frac{P}{N}$ 骤降
2. 玩家投入时间/劳动 → 换不来对等资产
3. 资产购买力过高 → 消费意愿低迷
4. 交易手续费萎缩 → 三路分账枯竭 → 暗域断血

这是典型的 **「生产力陷阱」**——越多人参与，单个人的产出越低，最终驱离参与者。

## 五、金融优化建议

### 5.1 优化一：引入信用评级平滑机制 —— 延期准备金拨备

**现状问题**：
递归追偿在瞬间全额扣除担保人所有 Bond，引发多米诺骨牌效应。

**优化方案**：引入基于历史纪元 (Epoch) 表现的**延期准备金拨备 (Deferred Provisioning)** 制度。

```
当前机制 (一刀切)              优化后 (递级缓冲)
┌──────────────────┐      ┌──────────────────────────────┐
│ 违约 → 瞬间全额   │      │ 违约 →                        │
│ 扣除担保人Bond    │      │                               │
│                  │      │ [Epoch T] 识别违约             │
│                  │      │    ↓                          │
│                  │      │ [Epoch T+1] 冻结担保人 30%     │
│                  │      │    ↓  观察期内担保人可申诉     │
│                  │      │ [Epoch T+2] 若确认为恶意       │
│                  │      │    → 递进扣除至 70%            │
│                  │      │    ↓                          │
│                  │      │ [Epoch T+3] 全额扣除 +         │
│                  │      │   上报递归追偿链               │
│                  │      │                               │
│                  │      │ 效果: 为关联节点提供            │
│                  │      │ 3个纪元的缓冲窗口              │
│                  │      │ 进行资产隔离和申诉              │
└──────────────────┘      └──────────────────────────────┘
```

**实现要素**：

| 参数 | 说明 | 建议值 |
|:---|:---|:---|
| `observation_epochs` | 观察期 Epoch 数 | 3 |
| `freeze_ratio_per_epoch` | 每 Epoch 递进冻结比例 | [30%, 70%, 100%] |
| `appeal_window` | 申诉窗口 (区块数) | 10,000 blocks |
| `honest_node_immunity` | 历史无违约节点的豁免阈值 | 连续 50 Epoch 无违约 |

**金融学依据**：类比巴塞尔 III 的**逆周期资本缓冲 (Counter-Cyclical Capital Buffer)**——不是取消惩罚，而是将惩罚从瞬时脉冲展开为时间序列，给市场足够的反应时间。

### 5.2 优化二：Inkwell 衍生品过桥 —— 影子凭证流通市场

**现状问题**：
Inkwell 1~90 天的资金锁定造成极高的机会成本和流动性折价。

**优化方案**：允许用户在明域发行**带 ZK 证明的远期影子凭证 (Shadow Claims)**，作为流动性折现工具在二级市场转让。

```
当前 (锁死)                     优化后 (流转)
┌──────────────────┐      ┌──────────────────────────────┐
│                  │      │                               │
│  暗域 Inkwell    │      │  暗域 Inkwell                │
│  ┌────────────┐  │      │  ┌────────────┐              │
│  │ 锁仓 1000  │  │      │  │ 锁仓 1000  │              │
│  │ 剩余 60天  │  │      │  │ 剩余 60天  │              │
│  └────────────┘  │      │  └─────┬──────┘              │
│                  │      │        │                      │
│  持有者:          │      │  生成 Shadow Claim           │
│  等 60 天        │      │  (ZK 绑定: 所有权+释放日)     │
│  无法动用         │      │        │                      │
│                  │      │        ↓                      │
│                  │      │  ┌────────────┐              │
│                  │      │  │ 明域二级市场│              │
│                  │      │  │            │              │
│                  │      │  │ Shadow Claim              │
│                  │      │  │ 面值: 950  │  ← 流动性折价 │
│                  │      │  │ 立即转让   │              │
│                  │      │  │            │              │
│                  │      │  │ 买方:       │              │
│                  │      │  │ 折价获得    │              │
│                  │      │  │ 60天后兑付  │              │
│                  │      │  └────────────┘              │
│                  │      │                               │
│                  │      │  效果:                         │
│                  │      │  · 卖方获得即时流动性           │
│                  │      │  · 买方获得时间套利             │
│                  │      │  · 系统释放锁仓资金效率         │
└──────────────────┘      └──────────────────────────────┘
```

**实现要素**：

| 参数 | 说明 | 约束 |
|:---|:---|:---|
| `shadow_claim` | 影子凭证 (ZK 绑定所有权) | 不可伪造、不可双花 |
| `maturity_date` | 兑付日期 | 与原始 Inkwell 释放日一致 |
| `face_value` | 面值 (= Inkwell 锁仓额) | 链上自动生成 |
| `market_price` | 二级市场交易价格 | 由市场供需决定，自然形成折价 |
| `transfer_proof` | ZK 所有权转移证明 | 不暴露原持有人身份 |

**ZK 电路约束** (`zkcircuits/shadow_claim/` 变体)：
1. 证明凭证对应的 Inkwell 锁仓真实存在且未过期
2. 证明转让人是该锁仓的合法受益人
3. 证明凭证未被双花 (nullifier 机制)
4. 不暴露原持有人身份 (与暗域隐私模型一致)

**金融学依据**：类比传统金融的**资产支持商业票据 (ABCP)** 和**远期利率协议 (FRA)**——将非流动性资产 (Illiquid Asset) 通过标准化凭证转化为可交易标的，释放资金效率。

**风险控制**：
- 单笔锁仓最多生成 1 张 Shadow Claim (防止过度杠杆化)
- Shadow Claim 的兑付优先级 ≥ 原始受益人
- 若原始 Inkwell 因系统级事件被撤销，Shadow Claim 自动失效 (风险由买方承担)

## 六、金融分析总结

### 6.1 系统定位

`nekop2p` 在金融工程上是一个 **高风险、高收益、具备强内生风控的闭环网络**：

| 维度 | 评估 |
|:---|:---|
| **资本效率** | ✅ 显著高于传统 DeFi——社交资本替代超额抵押 |
| **信用创造** | ✅ 具备商业银行级乘数效应——担保链衍生流动性 |
| **现金流可持续性** | ✅ 明域游戏解决冷启动 + 交叉补贴暗域基础设施 |
| **系统性风险** | ⚠️ 递归追偿在极端事件中是风险放大器而非缓冲器 |
| **资金效率** | ⚠️ Inkwell 延迟锁仓造成隐形折价和机会成本 |
| **通缩风险** | ⚠️ 掉落算法全局上限可能在增长期形成生产力陷阱 |
| **隐私-效率权衡** | ✅ 混沌池三锁提供理论不可追踪性，代价是时间成本 |

### 6.2 与传统金融的类比映射

| 传统金融概念 | nekop2p 对应 | 差异 |
|:---|:---|:---|
| 法定准备金 | 担保质押金 (Bonds) | 去中心化、自动执行、递归穿透 |
| 信用货币创造 | 担保链乘数效应 | 社交图替代央行、邓巴数硬约束 |
| 影子银行 | 暗域匿名借贷 | ZK 隐私 + 混沌池反追踪 |
| 优先级分层 (Tranching) | 递归追偿链 | 担保人承担多级风险吸收 |
| 交叉补贴 | 明域→暗域抽成注入 | 以丰补歉、内生循环 |
| 逆周期资本缓冲 | 延期准备金拨备 (优化一) | 从瞬时惩罚变为时间序列 |
| 资产证券化 (ABCP) | Shadow Claims (优化二) | 将非流动性锁仓变为可交易凭证 |

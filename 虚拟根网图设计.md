## 一、概念总览

### 1.1 什么是虚拟根网图

**虚拟根网图 (Virtual Root Graph)** 是一个不存在于任何单一物理服务器上、而是由历史共识和数学约束共同维持的高维影子网络。它将物理世界中因网络分裂、地理阻隔或应用隔离而失联的「孤立社区」，在数学空间中抽象为可计算拓扑距离的虚拟节点。

```
┌─────────────────────────────────────────────────────────────────────────┐
│                         虚拟根网图 (高维影子网络)                         │
│                                                                          │
│    ┌──────────┐                          ┌──────────┐                   │
│    │ 社区 V_A │──── 外交通道 (TrustEdge) ──│ 社区 V_B │                   │
│    │ Namespace│  拓扑距离 = 创世邀请链距    │ Namespace│                   │
│    │  0xAA   │   + 跨域担保质押额          │  0xCC   │                   │
│    └────┬─────┘                          └────┬─────┘                   │
│         │                                      │                         │
│         │    ┌──────────────┐                  │                         │
│         └────│  摆渡节点 F  │──────────────────┘                         │
│              │ (物理移动)    │                                            │
│              │ 携带 Shadow   │                                            │
│              │ Receipt + ZK  │                                            │
│              └──────────────┘                                            │
│                                                                          │
│  ═══════════════════════════════════════════════════════════════════════ │
│                        物理世界 (底层 P2P 网络)                           │
│  ┌─────────┐    ~~~ 间歇链路 ~~~    ┌─────────┐    ~~~ 物理移动 ~~~     │
│  │ 孤网 A  │ ←───────────────→ 摆渡 │ 孤网 B  │ ←───────────────→ ...  │
│  │(无外网) │   蓝牙/WiFi/LAN       │(无外网) │    USB/卫星/U盘         │
│  └─────────┘                       └─────────┘                          │
└─────────────────────────────────────────────────────────────────────────┘
```

### 1.2 三层抽象

虚拟根网图在不同尺度上有三种具象化形态：

| 层级 | 抽象粒度 | 核心机制 | 适用场景 |
|:---|:---|:---|:---|
| **微观层** | 个体 ↔ 个体 | ZK 凭证 + 哈希时间锁 + 树状祖先追溯 + 乐观博弈 | 双孤立个体直接交易 |
| **中观层** | 社区 ↔ 社区 | 社区虚拟化身 + 摆渡节点 + 纪元绑定影子收据 | 孤网社区间异步清算 |
| **宏观层** | 应用生态 ↔ 应用生态 | 命名空间隔离 + 跨域摆渡人 + 混沌池价值通兑 | 独立应用生态间跨域交易 |

## 二、微观层：双孤立个体的树状信任清算

### 2.1 问题定义

Alice 和 Bob 分属两个彼此断连的社区，拓扑距离 $>\infty$，没有共同在线担保人。Alice 想向 Bob 支付，但：
- 无法查询对方的链上状态
- 无法验证对方是否双花
- 无法执行在线追偿

### 2.2 三阶段协议

```
阶段1: ZK承诺生成             阶段2: 局部信任推演           阶段3: 网络重组清算
┌──────────────────┐      ┌──────────────────┐      ┌──────────────────┐
│ Alice 本地:       │      │ P2P 直连:        │      │ 大网恢复:         │
│                  │      │                  │      │                  │
│ master_secret    │      │ Noise_IK 握手    │      │ ABCI 胶水层      │
│   ↓              │      │   ↓              │      │   ↓              │
│ gnark ZK Proof   │      │ 树状祖先追溯     │      │ ZK 验证          │
│  "我是合法节点"   │ ───→ │ "有共同祖先?"    │ ───→ │ 双花检测         │
│  "未超信用额度"   │      │   ↓              │      │   ↓              │
│   ↓              │      │ 校验凭证链       │      │ 原子结算         │
│ BLAKE3(secret)   │      │ (轻量区块头)    │      │ 或               │
│   = H            │      │   ↓              │      │ 递归追偿         │
│   ↓              │      │ Bob 接受收据     │      │ (连坐担保人)     │
│ 发送 H + Proof   │      │ 本地缓冲队列     │      │                  │
│ 给 Bob           │      │ 提供服务/商品    │      │                  │
└──────────────────┘      └──────────────────┘      └──────────────────┘
```

#### 阶段1：可追溯信誉支票与 ZK 承诺

Alice 使用暗域主密钥 `master_secret` 结合历史有效信誉额度，在本地利用 `gnark` 框架生成**离线支付 ZK 证明**。

**电路硬约束** (`zkcircuits/repay/` 变体)：
1. 证明 Alice 是由创世/合法邀请链衍生出的树状网络节点
2. 拥有 $\ge 3$ 个历史担保人的 Ed25519 凭证链
3. 当前支付金额 $\le$ 历史信用资产总额

**秘密揭示锁**：Alice 生成 $H = \text{BLAKE3}(\text{secret})$，将资金锁定在哈希锁中，$H$ 与 ZK 证明发送给 Bob。

#### 阶段2：局部动态拓扑与信任推演

- Bob 通过 Noise_IK 握手与 Alice 建立临时 0.5-RTT 安全通道
- Bob 不依赖全局注册表，通过本地轻量同步状态校验 Alice 凭证中的**树状担保前缀**
- **若双方在树状结构上有共同祖先**（例如 3 个月前未分裂的同网节点）：Bob 通过历史区块头（~50MB）验证支票合法性，达成局部合意

#### 阶段3：网络重组的乐观清算

- Bob 的交易收据进入本地缓冲队列（类似 `Inkwell` 混沌池）
- 网络恢复后，Bob 节点立即将离线凭证广播给 Cosmos 链 ABCI 胶水层
- **正常情况**：链上验证 ZK 证明通过，原子化扣除 Alice 资产并转移给 Bob
- **双花情况**：`zkcircuits/marker/` 电路硬约束瞬间识别双花行为
- **递归惩罚**：Alice 资产清零 → 不足则扣除上级担保人质押金 → 递归上溯直至覆盖坏账

### 2.3 无历史交集的双孤立个体

若 Alice 和 Bob 来自两个**自创世以来无任何交集**的树状社区，纯密码学离线交易不可能（无共同信任根）：

**社会学博弈兜底**：
1. **第三方孤立实体担保**：寻找物理上双方都能接触的 Charlie（同时持有两社区凭证），充当临时跨树桥接人
2. **游戏化信用过桥**：利用明域资产（装备/金币）的透明状态机，通过单向兑换桥将金融债务转译为游戏道具的物理割让

## 三、中观层：社区作为虚拟节点的宏观异步清算

### 3.1 核心抽象：社区虚拟化身

将孤立社区整体抽象为虚拟根网图上的单一节点，通过**联合门限签名**生成社区集合身份。

```
┌──────────────────────────────────────────────────────────────────┐
│  物理层                      │  虚拟根网图层                     │
│                              │                                   │
│  [孤网 A: 500节点]           │  V_A ──────────── V_B            │
│  [孤网 B: 300节点]           │   ↑                ↑              │
│  [孤网 C: 120节点]           │   │  拓扑距离 d(A,B) = 4         │
│                              │   │  (基于创世邀请链)             │
│  物理拓扑:                   │   │                               │
│  A ⋯⋯(断连)⋯⋯ B            │   │  安全系数 = f(d, 质押额)     │
│  A ──(通畅)── C             │   │                               │
│  B ⋯⋯(断连)⋯⋯ C            │   │  若 d ≤ 阈值:                │
│                              │   │   允许宏观清算               │
│                              │   │  若 d > 阈值:                │
│                              │   │   拒绝或要求额外质押          │
└──────────────────────────────────────────────────────────────────┘
```

### 3.2 摆渡节点协议

**摆渡节点 (Ferry Node)** 是连接孤立社区的物理媒介——任何在网络间物理移动并运行 `neko-node` 的设备。

```
       孤网 A                     摆渡节点 F                     孤网 B
    ┌──────────┐              ┌──────────────┐              ┌──────────┐
    │          │              │              │              │          │
    │ 集合节点 │──Noise_NK──→│ 下载所有      │              │          │
    │ V_A      │  匿名握手    │ PENDING_      │              │          │
    │          │              │ OUTBOUND      │              │          │
    │ 状态树   │              │ 收据快照      │              │          │
    │ Tree_A   │              │              │              │          │
    │          │              │ 存储在        │──Noise_NK──→│ 集合节点 │
    │          │              │ BadgerDB      │  注入数据    │ V_B      │
    │          │              │              │              │          │
    │ 标记:    │              │ 摆渡节点      │              │ 验证:    │
    │ PENDING_ │              │ 不知内容      │              │ 联合签名 │
    │ OUTBOUND │              │ (洋葱加密)   │              │ ZK证明   │
    │          │              │              │              │          │
    │          │              │ 赚取:         │              │ 标记:    │
    │          │              │ 按单小费      │              │ ACCEPTED_│
    │          │              │ (Wages)       │              │ INBOUND  │
    └──────────┘              └──────────────┘              └──────────┘
```

**协议流程**：

1. **状态标记**：A 社区集合节点在本地状态树中锁定 Alice 资产，生成**社区级影子收据**（含联合门限签名 + ZK 证明）
2. **状态传递**：摆渡节点接近 A 社区时，通过 Noise_NK 匿名握手打包下载所有 `PENDING_OUTBOUND` 收据快照
3. **状态接收**：摆渡节点物理移动至 B 社区附近，注入数据；B 社区验证联合签名和 ZK 证明
4. **虚拟根网图推演**：B 社区根据创世邀请链计算 $V_A$ 与 $V_B$ 的拓扑距离，安全系数达标后标记为 `ACCEPTED_INBOUND`

### 3.3 社区级双花防护

**纪元绑定与门锁标记**：

每次摆渡释放的收据强制绑定全球递增的**纪元号 (Epoch)**。`zkcircuits/marker/` 电路硬约束：
- 若 A 社区在同一纪元内向 B、C 两社区发送同一笔资产的影子收据
- 大网重组时两收据在 ABCI 胶水层碰撞 → 瞬间触发**电路冲突**

**根网拓扑降维惩罚**：

| 违规规模 | 惩罚范围 | 具体措施 |
|:---|:---|:---|
| 个体双花 | 个体 | 链上资产清零 + 递归追偿至担保人 |
| 社区级欺诈 | 整个社区 V_A | 社区根节点质押金全额罚没 (Slashing) |
| 系统性作恶 | 邀请链上级社区 | 当初为作恶社区担保的上级社区连带罚没 |

这种「连带责任」迫使孤立社区在断连期间严苛审计本地账目——任何欺诈都会导致整个社区在虚拟根网图上被抹去或降级。

## 四、宏观层：跨应用生态的多维通兑

### 4.1 问题定义：逻辑孤岛 vs 物理互通

```
┌─────────────────────────────────────────────────────────────────┐
│                     物理网络层 (统一 IPv6 P2P)                    │
│                                                                  │
│  ┌──────────────┐    ┌──────────────┐    ┌──────────────┐       │
│  │ 应用 A: 游戏  │    │ 应用 B: 社交  │    │ 应用 C: 金融  │       │
│  │ Namespace AA  │    │ Namespace BB  │    │ Namespace CC  │       │
│  │              │    │              │    │              │       │
│  │ 资产: 金币   │    │ 资产: 声望   │    │ 资产: 信用额度│       │
│  │ 状态机: 游戏 │    │ 状态机: 社交 │    │ 状态机: 金融 │       │
│  │ 担保链: 独立 │    │ 担保链: 独立 │    │ 担保链: 独立 │       │
│  └──────┬───────┘    └──────┬───────┘    └──────┬───────┘       │
│         │                   │                   │                │
│         └───────────────────┼───────────────────┘                │
│                             │                                    │
│              底层协议共享 (ABCI 胶水层 + 混沌池)                   │
│              但信任域完全隔离 → 逻辑孤岛                          │
└─────────────────────────────────────────────────────────────────┘
```

不同应用在**业务逻辑、公钥体系、账本命名空间**上彼此隔离——形成应用层面的绝对孤岛。但底层 P2P 网络是统一的，节点在同一个 IPv6 组网中相互转发加密帧。

### 4.2 解决方案架构

```
┌──────────────────────────────────────────────────────────────────┐
│                        虚拟根网图 (最高维)                        │
│                                                                   │
│   ┌─────────┐      外交通道        ┌─────────┐                   │
│   │ 游戏 V_AA│ ←─── TrustEdge ──→ │ 金融 V_CC│                   │
│   │ (自治域) │    (链上治理建立)    │ (自治域) │                   │
│   └────┬────┘                      └────┬────┘                   │
│        │                                │                         │
│        │    ┌─────────────────┐         │                         │
│        └────│ 跨域摆渡节点 F   │─────────┘                        │
│             │ (多栖节点)       │                                  │
│             │ 同时运行 AA+CC   │                                  │
│             │ 的 neko-node     │                                  │
│             └────────┬────────┘                                  │
│                      │                                            │
│           ┌──────────┴──────────┐                                 │
│           │   暗域混沌池 Inkwell │                                 │
│           │  · 金额拆分 (3~7片) │                                 │
│           │  · 时间延迟 (1~90天)│                                 │
│           │  · 来源抹除         │                                 │
│           │  · 跨域价值通兑     │                                 │
│           └─────────────────────┘                                 │
└──────────────────────────────────────────────────────────────────┘
```

#### 4.2.1 命名空间隔离

底层 ABCI 胶水层为每个应用分配唯一 `NamespaceID`：

```go
// 每个应用在链上拥有独立的命名空间
type NamespaceID uint16
const (
    NamespaceGame    NamespaceID = 0xAA  // 游戏 A
    NamespaceSocial  NamespaceID = 0xBB  // 社交 B
    NamespaceFinance NamespaceID = 0xCC  // 金融 C
)

// 各命名空间的状态树完全隔离
// 一个命名空间的信用违约不会污染另一个的本地状态
```

#### 4.2.2 外交通道

两个社区的顶级记录节点通过链上治理建立**跨命名空间信任边 (Inter-Namespace Trust Edge)**：

- 社区 A 整体为社区 B 的底层密码学有效性做「宏观担保」
- 外交通道建立时双方需要质押 Bond（保证金）
- 拓扑距离 = 创世邀请链距 + 跨域担保质押额的函数

#### 4.2.3 混沌池通用清算

不同应用的资产（金币、声望、信用额度）形态不同，需要**降维至暗域**进行统一结算：

```
明域资产 (不可直接互换)              暗域通用积分             目标明域资产
┌──────────────────┐           ┌──────────────┐           ┌──────────────┐
│ 游戏A: 100金币    │──桥接──→│              │──桥接──→│ 金融C:        │
│                  │  bridge  │  混沌池       │  bridge  │ 等值信用额度   │
│                  │          │  (通用匿名     │          │              │
│ 社交B: 500声望   │──桥接──→│   结算积分)    │──桥接──→│              │
│                  │          │              │          │              │
│                  │          │ · 金额拆分    │          │              │
│                  │          │ · 时间延迟    │          │              │
│                  │          │ · 来源抹除    │          │              │
└──────────────────┘           └──────────────┘           └──────────────┘

Alice(游戏A) ──→ 锁定游戏资产 ──→ ZK证明 ──→ Inkwell ──→ 金融C释放信用 ──→ Bob(金融C)
                                         ↑
                                   跨域摆渡节点
                                   (物理路由)
```

**隐私保证**：金融 C 的节点 $V_C$ 收到暗域积分时，无法得知来源是游戏 A 的 Alice。ZK 电路仅证明「该价值由虚拟根网图上某个合法自治社区足额锁定且未双花」，微观身份被完全遮蔽。

### 4.3 宏观红利

| 特性 | 机制 | 效果 |
|:---|:---|:---|
| **隐私隔离** | 暗域混沌池 + ZK 遮蔽 | Alice 的游戏/社交记录绝不泄露给金融审计员 |
| **风险隔离** | Namespace 独立状态树 | 某应用崩盘/被黑只触及自身外交通道，不污染其他应用 |
| **流量混淆** | 统一 P2P 物理层 | 游戏高频流量为金融敏感交易提供天然防护垫 |
| **生态可插拔** | 外交通道链上治理 | 新应用通过质押 Bond + 建立 TrustEdge 即可接入通兑网络 |

## 五、密码学与博弈论的统一

### 5.1 信任根追溯链

所有交易——无论微观个体、中观社区还是宏观应用——最终都可以追溯到创世邀请链：

```
创世节点
  ├── 邀请 A → 邀请 A₁ → 邀请 A₂ → Alice
  │                              └── 此链形成 Alice 的信用根
  ├── 邀请 B → 邀请 B₁ → Bob
  │
  └── 任何社区/应用的信任根都可沿此树向上追溯
     虚拟根网图上两点距离 = 最低共同祖先 (LCA) 到各自节点的步数和
```

### 5.2 欺骗成本的递归放大

```
欺骗类型          检测时机        惩罚范围          威慑力来源
─────────────────────────────────────────────────────────────
个体双花          网络重组瞬间    个体→担保人       资产清零 + 社会关系破裂
社区级欺诈        大网恢复后      整个社区根节点    社区质押金全额罚没
应用生态作恶      跨域审计触发    应用外交通道      被虚拟根网图除名
```

核心设计原则：**欺骗的获利是局部的（一个离线交易），但惩罚是递归的（连带整个信任链）。** 这使任何理性参与者在孤立状态下也不敢作恶——因为网络的恢复是概率性事件，而惩罚的确定性由密码学保证。

## 八、设计总结

虚拟根网图将 `nekop2p` 从「去中心化城邦」升级为「承载无数逻辑孤立文明共同繁荣的星际网络基础设施」：

1. **物理世界的移动弥补数字世界的断连** — 摆渡节点用物理穿梭替代实时在线共识
2. **数学空间的递归惩罚对冲离线欺诈风险** — 密码学保证惩罚的确定性使欺骗得不偿失
3. **暗域混沌池润滑跨域生态** — 价值互通的同时彻底遮蔽微观身份
4. **Namespace 隔离保证风险不扩散** — 每个应用生态是独立自治域，崩盘不污染他者

**交易不再依赖微观个体的实时在线，而是变成了孤立时空之间的宏观状态对齐。**

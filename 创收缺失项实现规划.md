## 1️⃣ 借贷利息系统 (优先级最高)

### 现状
- `SeedLoanReserve` 有储备金但**无利息计算**
- 借贷 = 纯信用分配，不产生利息收入
- 担保人承担违约风险但**无利息收益**

### 实现方案

#### 1.1 类型扩展 (`x/brightchain/types/`)

```go
// Pool 添加利息子账户
type Pool struct {
    // ... 现有字段 ...
    InterestEarned   uint64  // 累计利息收入
    InterestReserve  uint64  // 利息储备金
}

// LoanRecord 添加利率字段
type LoanRecord struct {
    // ... 现有字段 ...
    InterestRate    uint64  // 年化利率 (基点, 500=5%)
    InterestAccrued uint64  // 累计利息
    LastAccrualAt   int64   // 上次计息时间
}

// EconomyParams 添加利率参数
type EconomyParams struct {
    BaseInterestRate    uint64  // 基础利率 (基点), 默认 500
    RiskPremiumMin      uint64  // 最低风险溢价, 默认 100
    RiskPremiumMax      uint64  // 最高风险溢价, 默认 1000
    InterestDistribution [3]uint64 // [SeedLoan%, Guarantor%, Community%]
}
```

#### 1.2 利率计算 (`x/brightchain/keeper/`)

```
利率 = 基础利率 + 风险溢价
风险溢价 = (1000 - creditScore) / 1000 × (RiskPremiumMax - RiskPremiumMin) + RiskPremiumMin

利息 = 本金 × 利率 × (当前时间 - 上次计息时间) / 年秒数
```

#### 1.3 利息分账

```
还款利息收入 → 分三路:
  50% → SeedLoanReserve (再借贷)
  25% → 担保人收益 (Bond 利息分红)
  25% → Community (公共物品)
```

#### 1.4 修改文件清单

| 文件 | 改动 | 工作量 |
|:---|:---|:---|
| `types/types_v2.go` + `types.go` | Pool + LoanRecord + EconomyParams 扩展 | 0.5h |
| `keeper/keeper.go` + `keeper_v2.go` | AccrueInterest + CollectInterest 方法 | 2h |

## 2️⃣ 动态手续费率 (优先级高)

### 现状
- 分账比例硬编码: `25/25/20/15/15`
- 无治理调节机制

### 实现方案

#### 2.1 参数化 (`x/brightchain/types/`)

```go
type FeeParams struct {
    SalaryRelayRate     uint64  // 中继节点比例 (%)
    SalaryRecordRate    uint64  // 记录节点比例 (%)
    SeedLoanRate        uint64  // 种子借贷比例 (%)
    BadDebtRate         uint64  // 坏账准备金比例 (%)
    CommunityRate       uint64  // 社区基金比例 (%)
    DynamicEnabled      bool    // 是否启用动态费率
    CongestionThreshold uint64  // 拥堵阈值 (pending tx count)
    CongestionPremium   uint64  // 拥堵附加费率 (%)
}

// 默认值
func DefaultFeeParams() FeeParams {
    return FeeParams{
        SalaryRelayRate:  25,
        SalaryRecordRate: 25,
        SeedLoanRate:     20,
        BadDebtRate:      15,
        CommunityRate:    15,
        DynamicEnabled:   false,
    }
}
```

#### 2.2 修改 Collector

```go
func (k *Keeper) CollectFees(ctx, feeAmount uint64, params FeeParams) error {
    // 动态费率: 如果 pending tx > threshold, 增加社区基金比例
    if params.DynamicEnabled && pendingCount > params.CongestionThreshold {
        // 拥堵时社区基金多收 premium%
        congestionFee := feeAmount * params.CongestionPremium / 100
        pool.Community += congestionFee
        feeAmount -= congestionFee
    }
    // 按参数比例分配
    pool.SalaryRelay += feeAmount * params.SalaryRelayRate / 100
    // ...
}
```

#### 2.3 治理集成

- 通过 `EconomyGovernor.ProposeChange("fee_params", newParams)` 提案
- 30天渐变过渡期
- EndBlocker 中检查到期提案并应用

## 3️⃣ 跨域消息费 (优先级中)

### 现状
- VRG `ShadowReceipt` 跨域桥接完全免费
- 摆渡节点承担转发成本但无补偿

### 实现方案

#### 3.1 计费模型

```go
// x/vrg/types (新增)
type CrossDomainFee struct {
    BaseFee       uint64  // 基础费用 (per message)
    DataFeeRate   uint64  // 数据费率 (per byte)
    VerifyFee     uint64  // ZK 验证费 (per proof)
}

// 费用计算
func CalcCrossDomainFee(receipt ShadowReceipt) uint64 {
    dataSize := len(receipt.LockTxHash) + len(receipt.ReceiptID)
    return BaseFee + DataFeeRate * dataSize + VerifyFee
}
```

#### 3.2 费用分账

```
跨域消息费 → 分三路:
  60% → 摆渡节点 (Ferry Node 激励)
  25% → 源社区 Community 基金
  15% → 目标社区 Community 基金
```

## 5️⃣ 游戏金币市场汇率 (优先级低)

### 现状
- 汇率固定: `GoldPerRecharge = GoldPerWithdraw = 100`
- 无法响应供需变化

### 实现方案

```go
type ExchangeParams struct {
    GoldPerRecharge   float64  // 充值汇率 (治理可调)
    GoldPerWithdraw   float64  // 提现汇率 (略高于充值, 0.5% 价差)
    SpreadRate        float64  // 买卖价差 (默认0.5%)
    ReserveRatio      float64  // 储备金率 (发行金币的背书比例)
}

// 自适应汇率
func CalcAdaptiveRate(reserve, circulation uint64) float64 {
    // 储备率 = reserve / circulation
    // 储备率低 → 提现汇率降低 (防止挤兑)
    // 储备率高 → 汇率稳定
    baseRate := 100.0
    reserveRatio := float64(reserve) / float64(circulation)
    return baseRate * (0.8 + 0.4*reserveRatio)  // 80-120% 浮动
}
```

## 四、数据流：一笔借贷的完整路径

```
用户A (钱包界面)
  │
  ├─① 暗链借贷申请 ──────────────────────────────────────────────┐
  │  localapi.SendMessage → node.SendAnonMessage(暗道)           │
  │  → onion路由(3跳) → 暗链节点                                 │
  │  → x/darkchain MsgRequestLoan {anon_id, zk_credit_proof}    │
  │  → zkcircuits/credit 验证票据未双花                           │
  │  → zkcircuits/marker 门锁检查(同区块不重复)                   │
  │  → 暗链记录 LoanRecord                                       │
  │                                                               │
  ├─② 明链放款 ──────────────────────────────────────────────────┤
  │  用户提交 zkcircuits/identity 证明                            │
  │  → x/brightchain 验证 ≥3担保人 + 信用额度                     │
  │  → 资金池划款至托管                                           │
  │                                                               │
  ├─③ 暗链结算 ──────────────────────────────────────────────────┤
  │  双方确认交付 → MsgSettleInstruction                          │
  │  → 明链划转托管资金                                           │
  │                                                               │
  └─④ 明链还款 (语义翻转: 向资金池放贷) ─────────────────────────┤
     inkwell 三锁:
       锁1: 时间随机化 → scheduler.go 决定还款日 (1~90天)
       锁2: 金额拆封 → inkwell.go 拆成3~7片随机碎片
       锁3: 来源伪装 → 可选接力节点代付
     → x/brightchain MsgRepay {zk_repay_proof, fragments}
     → 递归追偿检查 (如果违约)
```

## 六、密码学算法选型

| 用途 | 算法 | 标准化程度 | 源码位置 |
|:---|:---|:---|:---|
| 密钥封装 (KEM) | ECDH (Curve25519) | RFC 7748 | `crypto/kem.go` |
| 对称加密 | AES-256-GCM / ChaCha20-Poly1305 | NIST SP 800-38D | `frame/frame.go`, `noise/handshake.go` |
| 签名 | Ed25519 | RFC 8032 | `crypto/sign.go` |
| 哈希 | BLAKE3 | 广泛使用 | 各包引用 |
| 后量子 (身份密钥) | Kyber-1024 | NIST PQC 标准 | `crypto/keys.go` |
| 零知识证明 | zk-SNARKs (Groth16) | gnark 成熟实现 | `zkcircuits/` |
| 双棘轮 | Signal Protocol 变体 | 广泛部署 | `ratchet/ratchet.go` |
| 握手协议 | Noise IK/NK | Noise Framework | `noise/handshake.go` |

│                    暗链 (债务链/交易层)               │

## 十、传输帧格式

```
线上字节 (节点间 TCP 连接):
┌────────┬──────────┬──────────────────────┬──────────┐
│  Len   │  Nonce   │      Ciphertext      │ Auth Tag │
│ (2B)   │  (8B)    │      (variable)      │  (16B)   │
│ 大端序  │ 递增计数器 │ ChaCha20-Poly1305加密 │          │
└────────┴──────────┴──────────────────────┴──────────┘
                        ↓ 解密后
┌────────┬────────┬─────────┬───────────┬──────────────┐
│  Ver   │  Type  │  Flags  │ Stream ID │   Payload    │
│ (1B)   │  (1B)  │  (1B)   │  (2B)     │  (variable)  │
└────────┴────────┴─────────┴───────────┴──────────────┘

帧类型:
  0x01 BEACON  — 加密信标转发
  0x02 DATA    — 端到端加密消息 (双棘轮后)
  0x03 PING    — 心跳保活
  0x04 PONG    — 心跳应答
  0x05 CLOSE   — 优雅关闭
  0x06 ROUTE   — 洋葱路由中继
```
